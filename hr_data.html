แผนผังการปรับปรุงคุณภาพข้อมูลพนักงาน

(HR Data Quality Improvement Canvas)
การไฟฟ้าส่วนภูมิภาค (กฟภ.)

สำรวจแผนงาน

1. ปัญหาหลักและข้อจำกัดปัจจุบัน

นี่คือสถานการณ์และความท้าทายที่เรากำลังเผชิญอยู่ในการบริหารจัดการข้อมูล HR :

การกรอกข้อมูลที่ระบบ SAP (SAP Data Entry)

  • - ขาดการตรวจสอบความถูกต้อง (Data Validation) : ระบบ SAP HR Module ไม่มี Data Validation ในขั้นตอนการป้อนข้อมูลพนักงาน ทำให้เกิดความผิดพลาดตั้งแต่จุดเริ่มต้น
  • - ข้อจำกัดทางสัญญา : ไม่สามารถเพิ่มฟังก์ชัน Data Validation หรือดำเนินการแก้ไขข้อมูล (Data Cleansing) บนระบบ SAP โดยตรงได้
  • - ไม่สามารถเขียนข้อมูลกลับ (No Write-Back) : ข้อมูลที่ถูกแก้ไขในแหล่งอื่นไม่สามารถเขียนกลับไปอัปเดตที่ SAP ได้

การกระจายตัวของแหล่งข้อมูล (Data Fragmentation)

  • - ปัจจุบันมีแหล่งข้อมูลพนักงาน 3 แหล่งหลัก ได้แก่ SAP, SQL Database (สำหรับกระบวนการ Non-SAP) และ Database ของแต่ละกระบวนการ Non-SAP
  • - การกระจายตัวนี้นำไปสู่ปัญหา ข้อมูลกระจัดกระจายไม่เชื่อมโยงกัน (Data Silo) และ ความไม่สอดคล้องของข้อมูล (Data Inconsistency) ระหว่างแหล่งต่าง ๆ

การแก้ไขปัญหาเชิงรับ (Reactive Problem Solving)

  • การรับรู้ปัญหาคุณภาพข้อมูลเกิดขึ้นเมื่อมีปัญหาในการใช้งานแล้วเท่านั้น ทำให้การแก้ไขเป็นแบบปลายเหตุ และไม่ยั่งยืนในระยะยาว

2. กลยุทธ์หลัก: การทวนสอบข้อมูล (Data Verification Method) สำหรับ SAP

นี่คือหัวใจสำคัญในการขับเคลื่อนคุณภาพข้อมูลอย่างยั่งยืนในระยะยาว :

กระบวนการ (Process)

  • - สกัดข้อมูล (Extract Data) : ดึงข้อมูลจาก SAP ออกมา
  • - ตรวจสอบและระบุ (Validate & Identify) : ใช้ กฎเกณฑ์ข้อมูล (Data Element Rule) เพื่อตรวจสอบและระบุข้อมูลที่ไม่มีคุณภาพ
  • - แสดงผลและชี้เป้า (Visualize & Pinpoint) : จัดทำแดชบอร์ด (Dashboard) เพื่อชี้เป้าข้อมูลที่ผิดพลาดและระบุผู้รับผิดชอบในการป้อนข้อมูล
  • - สร้างแรงจูงใจและบังคับใช้ (Incentivize & Enforce) : กำหนดตัวชี้วัดผลงานหลัก (KPI) ที่เชื่อมโยงกับการปรับปรุงคุณภาพข้อมูล เพื่อกระตุ้นและบังคับให้ผู้เกี่ยวข้องดำเนินการแก้ไขข้อมูลให้ถูกต้อง

วัตถุประสงค์ (Objectives)

  • - แก้ไขปัญหาที่ต้นตอทางอ้อม โดยการกระตุ้นให้ผู้ใช้งานปรับปรุงพฤติกรรมการป้อนข้อมูล
  • - สร้างความตระหนักรู้ด้านข้อมูล (Data Awareness) และส่งเสริมวัฒนธรรมองค์กรที่ขับเคลื่อนด้วยข้อมูล (Data-Driven Culture)

ผลลัพธ์ที่คาดหวัง (Expected Outcomes)

  • - ข้อมูลบน SAP มีคุณภาพดีขึ้นอย่างยั่งยืนในระยะยาว
  • - ผู้ใช้งานมีความตระหนักและรับผิดชอบ Data Quality มากขึ้น
  • - ข้อมูลที่ Replicated ไป SQL Database มีคุณภาพสูงขึ้นตาม

กลยุทธ์สนับสนุนและกลยุทธ์เสริม

การจัดการฐานข้อมูลอื่น ๆ เพื่อสนับสนุนการดำเนินงาน:

3. กลยุทธ์สนับสนุน: SQL Database

การจัดการฐานข้อมูล SQL เพื่อสนับสนุนกระบวนการ Non-SAP :

กระบวนการ (Process)

  • - ETL (Extract, Transform, Load): ดึงข้อมูลจาก SAP ไปยัง SQL Database
  • - Data Cleansing (ในขั้นตอน Transform หรือหลัง Load): ปรับปรุงคุณภาพข้อมูลด้วยสคริปต์ (Script) สำหรับกระบวนการ Non-SAP

วัตถุประสงค์ (Objectives)

  • - จัดหาข้อมูลคุณภาพดีสำหรับ Non-SAP Process
  • - ลดผลกระทบจากข้อมูล SAP ที่ยังไม่มีคุณภาพเต็มที่

ผลลัพธ์ที่คาดหวัง (Expected Outcomes)

  • - กระบวนการ Non-SAP ได้ใช้ข้อมูลที่มีคุณภาพดีขึ้น
  • - เป็นการแก้ปัญหาเฉพาะหน้าเพื่อประคองสถานการณ์

4. กลยุทธ์เสริม: Database ของกระบวนการ Non-SAP

การเสริมสร้างคุณภาพข้อมูล ณ จุดใช้งานปลายทาง :

กระบวนการ (Process)

  • - Owner Responsibility: สนับสนุนให้เจ้าของกระบวนการ Non-SAP ดำเนินการ Data Validation และ Data Cleansing ข้อมูลภายในระบบของตนเองก่อนการบันทึก

วัตถุประสงค์ (Objectives)

  • - เพิ่มคุณภาพข้อมูล ณ จุดใช้งานปลายทาง
  • - สร้างความรับผิดชอบของ Owner ข้อมูล

ผลลัพธ์ที่คาดหวัง (Expected Outcomes)

  • - ข้อมูลในระบบ Non-SAP มีความถูกต้องตามวัตถุประสงค์ของแต่ละกระบวนการ

5. บทสรุป

ในสภาวะที่มีข้อจำกัดในการปรับเปลี่ยนระบบหลักอย่าง SAP การดำเนินการตาม "วิธีการทวนสอบข้อมูล" ถือเป็นกลยุทธ์ที่สำคัญที่สุดและเป็นรากฐานในการยกระดับคุณภาพข้อมูลพนักงานของ กฟภ. อย่างยั่งยืนในระยะยาว โดยมี SQL Database และ Database ของกระบวนการ Non-SAP เป็นส่วนเสริมที่ช่วยให้กระบวนการทางธุรกิจสามารถดำเนินไปได้อย่างต่อเนื่องในขณะที่ดำเนินการปรับปรุงข้อมูลหลักให้ดีขึ้น การสร้างความตระหนักและการบังคับใช้ KPI จะเป็นกุญแจสำคัญในการขับเคลื่อนความสำเร็จของแผนงานนี้

เป้าหมายสูงสุด (Ultimate Goals)

  • - สร้าง Data Culture ที่แข็งแกร่ง
  • - ยกระดับ Data Quality ของข้อมูล HR โดยรวม
  • - สนับสนุนการตัดสินใจทางธุรกิจด้วยข้อมูลที่น่าเชื่อถือ

ผลลัพธ์สูงสุดที่คาดหวัง (Ultimate Expected Outcomes)

  • - องค์กรมีข้อมูลพนักงานที่สะอาด ถูกต้อง และน่าเชื่อถือมากขึ้น
  • - ลดความเสี่ยงจากการตัดสินใจผิดพลาดที่เกิดจากข้อมูล
  • - สร้างรากฐานที่ดีสำหรับการนำ AI/ML มาใช้ในอนาคต
