บทบาทหน้าที่ & ข้อตกลง (Roles & SLA)

การแบ่งงานและการส่งต่องานที่ชัดเจน

ฝ่ายกำกับดูแลข้อมูล (ฝดข.)

The Gatekeeper
  • Phase 1: รับเรื่อง, ตรวจสอบกฎหมาย/PDPA, ประสาน Data Owner, เสนอ CDO อนุมัติ
  • Phase 2: ขึ้นทะเบียน Metadata บน Catalog, ทำทะเบียนผู้ขอใช้ข้อมูล
  • Key Output: "บันทึกแจ้งอนุมัติ" ส่งให้ ฝปด.

Handshake Protocol (SLA)

1. แจ้งผลอนุมัติ (ฝดข. > ฝปด.) 3 วัน
2. แจ้งผลเชื่อมต่อ (ฝปด. > ฝดข.) 5-10 วัน
3. รายงานการใช้ (ฝปด. > ฝดข.) ทุกวันที่ 5
*เป้าหมายเวลาทำการ (Business Days)

ฝ่ายปฏิบัติการข้อมูล (ฝปด.)

The Builder
  • Phase 1: รอใบแจ้งอนุมัติ, พัฒนา API, กำหนดสิทธิ์ Access
  • Phase 2: เชื่อมต่อระบบ, ทดสอบ (UAT), แจ้ง Go-Live
  • Phase 3: Monitor ระบบ, เก็บ Log, สรุป Report ส่งคืน ฝดข.

แผนการดำเนินงานเร่งด่วน (Next Steps)

1. Cataloging

จัดทำรายละเอียด Metadata ของชุดข้อมูล API ทั้ง 4 ชุด ให้ครบถ้วนสมบูรณ์

2. Publishing

ประกาศขึ้น PEA Data Catalog และส่ง Metadata 3 ชุดไปยัง GD Catalog

3. Governance Setup

กำหนดกระบวนการพิจารณาอนุญาตให้ชัดเจน และสื่อสารกับ Data Owner

4. Technical Prep

เตรียม API และระบบ Monitoring ให้พร้อมรองรับ Log จากทั้ง 2 ช่องทาง
